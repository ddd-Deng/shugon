## 版本 0.3.3 - 敌人 cooldown 修复 (2026-03-31)

### 本次修改的文件

| 文件 | 改动内容 |
|------|---------|
| `scripts/battle/enemy_base.gd` | 修复 cooldown 未生效问题：敌人仅在倒计时归零时行动，非行动回合强制待机；意图显示加入待机剩余回合 |

### 修复内容

- 原问题：敌人每回合都执行 `decide_action()` 并立刻行动，`action_cooldown` 变量没有真正参与执行流程。
- 新增 `_consume_turn_and_check_can_act()`：敌人回合开始先消耗 1 点倒计时，只有倒计时归零才允许行动。
- `execute_action()` 现在会先检查 cooldown：
  - 未就绪：本回合直接待机，不执行移动/攻击。
  - 已就绪：执行 AI 行动，并重置倒计时。
- `update_intent()` 与 `Intent.IDLE` 显示改为基于 cooldown：
  - 非行动回合显示 `待机(剩余回合)`，便于肉眼验证节奏。
  - 行动回合才预告真实意图（攻击/移动/技能）。
- 增加容错：`action_cooldown` 最小为 `1`，避免非法配置。

### 结果

- 敌人不再每回合都攻击；默认史莱姆 `cooldown=2` 时表现为隔回合行动一次。

## 版本 0.3.1 - 敌人回合节奏优化 + 额外命中反馈 (2026-03-31)

### 本次修改的文件

| 文件 | 改动内容 |
|------|---------|
| `scripts/battle/battle_scene.gd` | 新增敌人回合开始延时、敌人逐个行动间隔、敌人回合结束延时；新增 Enemy Turn 提示动效；新增命中脉冲与轻微屏幕震动 |
| `scripts/battle/entity.gd` | 新增 `damaged` 信号，用于将受击事件传递给战斗场景特效系统 |

### 新增/调整内容

- **敌人回合延时**：玩家行动结束后，敌人不会立刻行动，先有短暂缓冲。
- **逐个敌人行动节奏**：敌人之间加入间隔，不再一帧内全部执行。
- **回合提示动效**：敌人阶段开始时，`Enemy Turn` 文本会有轻微弹动与淡入。
- **命中脉冲**：受击点会出现短促扩散的命中图形。
- **轻微屏幕震动**：命中时全场景横向小幅震动，增强打击感。

### 可调参数

- `enemy_turn_start_delay`：敌人回合开始前延时（默认 `0.35`）
- `enemy_action_interval`：敌人之间行动间隔（默认 `0.2`）
- `enemy_turn_end_delay`：敌人回合结束后延时（默认 `0.2`）

## 版本 0.3.0 - 战斗视觉反馈特效 (2026-03-31)

### 本次修改的文件

| 文件 | 改动内容 |
|------|---------|
| `scripts/battle/entity.gd` | 新增通用战斗特效：受击闪红、受击抖动、伤害飘字、击杀 `KO` 飘字、攻击前冲动画 |
| `scripts/battle/player.gd` | 玩家攻击前触发前冲特效（键盘攻击与鼠标点敌攻击都生效） |
| `scripts/battle/enemy_base.gd` | 敌人攻击前触发前冲特效 |

### 新增视觉反馈

- **攻击前冲**：攻击方在命中前短暂向目标方向前冲再回位。
- **受击闪红**：被命中单位会瞬间泛红后恢复。
- **受击抖动**：被命中单位会有短促左右抖动反馈。
- **伤害飘字**：命中时显示 `-1` 等红色飘字。
- **击杀飘字**：单位死亡时显示金色 `KO` 飘字。

### 结果

- 战斗动作的命中感与可读性明显提升，玩家可以更直观地感知“谁在攻击、谁受到了伤害、是否击杀成功”。

## 版本 0.2.1 - 悬停高亮修复 (2026-03-31)

### 本次修改的文件

| 文件 | 改动内容 |
|------|---------|
| `scripts/battle/battle_scene.gd` | 修复格子悬停颜色残留问题：同一格悬停时也会实时重算高亮；玩家/敌人行动后强制刷新整行格子颜色 |

### 修复内容

- 修复问题：鼠标悬停敌人后，移开鼠标或局面变化时，红色可攻击高亮有时不会及时恢复。
- 修改 `_set_hover_cell()` 逻辑：不再仅在“悬停格子变化”时更新颜色，同一格悬停也会重算可攻击状态。
- 新增 `_apply_hover_color()`：集中处理悬停格子的高亮规则（可攻击红色、可移动绿色、不可达灰色）。
- 新增 `_refresh_grid_visuals()`：在玩家行动完成、敌人行动完成、玩家输入阶段开始时，刷新所有格子底色并重新套用悬停高亮。

### 结果

- 悬停高亮会随局面变化即时刷新，不再需要移动鼠标才能恢复正确颜色。

## 版本 0.2.0 - 敌人意图 + 鼠标交互 (2026-03-31)

### 本次修改的文件

| 文件 | 改动内容 |
|------|---------|
| `scripts/battle/battle_scene.gd` | **大幅重写** — 新增鼠标点击移动/攻击、格子悬停高亮、Attack/Skip按钮、敌人意图刷新、战斗结束状态管理 |
| `scripts/battle/enemy_base.gd` | 新增意图显示系统：`update_intent()` 计算意图，头顶显示文字标签（ATK!/>>>/<<</.../!!!） |
| `scripts/battle/player.gd` | 新增 `try_move_to(pos)` 点击目标格子移动、`try_attack_at(pos)` 点击目标格子攻击 |
| `scripts/battle/grid.gd` | 新增 `world_to_grid()` 将鼠标坐标转换为格子索引 |

### 新增功能

#### 敌人意图预告
- 每个回合开始时，所有敌人头顶显示它们下一步的行为
- `ATK!` (红色) = 将要攻击
- `>>>` 或 `<<<` (黄色) = 将要移动
- `...` (灰色) = 待机
- 意图基于敌人当前与玩家的距离实时计算

#### 鼠标点击操作
- **点击相邻空格子** → 玩家移动到该格子（绿色高亮提示）
- **点击相邻敌人格子** → 玩家攻击该敌人（红色高亮提示）
- 非相邻格子点击无效（暗色高亮提示）
- 悬停时自动变色反馈可操作性

#### UI按钮
- **Attack 按钮** — 点击执行面朝方向攻击（等同 Space 键）
- **Skip 按钮** — 点击跳过回合（等同 S 键）
- 按钮在非玩家回合自动变灰禁用

### 当前完整操作方式
| 操作 | 键盘 | 鼠标 |
|------|------|------|
| 左移 | A / 左方向键 | 点击左边相邻空格子 |
| 右移 | D / 右方向键 | 点击右边相邻空格子 |
| 攻击 | Space / Enter | 点击相邻敌人 或 Attack按钮 |
| 跳过 | S | Skip按钮 |

### 接下来计划实现的功能（按优先级）
1. **战斗动画/视觉反馈** - 攻击闪烁、受伤抖动、移动平滑过渡
2. **技能系统框架** - 可配置的攻击/技能，替代当前的固定攻击
3. **更多敌人类型** - 不同 AI 行为模式（远程、冲锋、防御等）
4. **占位符像素素材** - 替换色块为简单像素画
5. **关卡/楼层系统** - 多场战斗的 Roguelike 流程
6. **奖励/升级界面** - 战斗间选择新技能

### 已知问题
- 格子悬停颜色在玩家移动后不会立即刷新（需要鼠标移动触发）
- 没有动画过渡，敌人行动是瞬间完成的
- Attack按钮只攻击面朝方向，如果面朝方向没有敌人则无效
- 没有音效和视觉特效

## 版本 0.1.0 - 项目骨架 (2026-03-31)

### 本次完成的工作

#### 项目配置
- `project.godot` - Godot 4.5 项目配置文件
  - 原生分辨率 384x216，窗口 1920x1080（5倍放大）
  - 像素风渲染设置（最近邻采样，无纹理过滤）
  - GL Compatibility 渲染器
  - 输入映射：A/D 移动，Space 攻击，S 跳过回合，方向键也可移动
  - GameManager 注册为全局 Autoload

#### 核心脚本
| 文件 | 作用 |
|------|------|
| `scripts/core/game_manager.gd` | 全局游戏管理器（Autoload），管理游戏状态、存档数据、场景切换 |
| `scripts/battle/turn_manager.gd` | 回合管理器，控制 玩家输入→玩家行动→敌人行动→结算 的流程 |
| `scripts/battle/grid.gd` | 1D 格子数据结构，管理实体位置、移动、碰撞检测 |
| `scripts/battle/entity.gd` | 实体基类，包含 HP、受伤、死亡等通用逻辑 |
| `scripts/battle/player.gd` | 玩家类，继承 entity，处理移动和基础攻击 |
| `scripts/battle/enemy_base.gd` | 敌人基类，继承 entity，包含 AI 决策框架（靠近/攻击） |
| `scripts/battle/enemies/slime_enemy.gd` | 史莱姆敌人（测试用），2HP，每2回合行动 |
| `scripts/battle/battle_scene.gd` | 战斗场景控制器，协调所有战斗逻辑和 UI |
| `scripts/ui/main_menu.gd` | 主菜单脚本，处理开始按钮 |
